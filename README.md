# DIKWP HumanBecoming Evolution OS

**DIKWP 人类生成与开放演化系统**

A GitHub-ready, offline-first reference implementation for rethinking human development as a multi-scale, self-maintaining, self-revising, environment-coupled process rather than a fixed biological, professional, or narrative object.

## Core proposition

A future human is not defined by a single substrate, role, credential, memory store, or biological snapshot. The system treats human continuity as a layered and revisable contract across body, cognition, affect, relationships, technology, institutions, ecology, and narrative.

It does **not** rank human worth, prescribe genetic or biomedical enhancement, or decide who counts as human.

## Quick start

### Standalone browser studio

Open `index.html` directly. No server, account, network, API key, or database is required.

### Python CLI

```bash
pip install -e .
humanbecoming analyze examples/sample_humanbecoming_case.json --out outputs/demo
humanbecoming guard "build a reversible human-AI memory portability experiment"
humanbecoming static-audit src --out outputs/demo/static_boundary_audit_report.json
```

## Safety and governance

The system is for voluntary learning, governance, personal development, and research prototyping. It must not be used for eugenics, coercive enhancement, human-worth scoring, employment/admission/insurance decisions, covert monitoring, clinical intervention, or rights reduction.

See `docs/governance_boundary.md`.
